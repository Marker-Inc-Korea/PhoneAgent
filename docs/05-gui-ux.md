# PhoneAgent — GUI / UX 기획

## 디자인 원칙 (QUALITY_DEFAULTS 준수)
- 색: 단일 dominant(딥 인디고) + accent(앰버) + 중성 스케일. Material3 dynamicColor 비활성, 직접 토큰.
- 타이포: 시스템 기본 대신 무게 대비(Display SemiBold / Body Regular). line-height 명시.
- 간격: 4/8dp 배수. radius 4/8/12 3단계.
- 모션: alpha/translation만. 150~300ms.
- 접근성: 터치 타깃 ≥48dp, contentDescription, 색만으로 상태 표현 금지(아이콘 병행).

## 네비게이션
하단 없음(단순). Onboarding(조건부) → Home(시작). Home 상단 앱바 우측 톱니 → Settings.
권한 미충족 시 Home 상단 경고 카드 → Onboarding/시스템 설정으로.

## 화면별 기획

### Onboarding (권한)
- 헤더: "PhoneAgent 시작하기"
- 카드1 접근성 권한: 상태칩(필요/완료) + "접근성 설정 열기" 버튼 → 시스템 설정.
- 카드2 알림 권한(API33+): "알림 허용" 버튼.
- 모두 완료 시 하단 "시작하기" 활성 → Home.

### Home (메인)
- 앱바: "PhoneAgent" + 설정 아이콘.
- 권한/키 미충족 경고 카드(있을 때만).
- 작업 입력: 멀티라인 TextField, placeholder "무엇을 해드릴까요? 예: Gmail에서 새 메일 확인해줘",
  하단 예시 칩 3개(탭하면 입력 채움): [Gmail 새 메일 요약] [설정 화면 열기] [화면 읽어줘].
- 실행 버튼(Primary, 전체폭). 실행 중이면 "중지"로 토글.
- 진행 영역: 실행 중 단계 로그 리스트(스텝#, thought 요약, action, 결과 아이콘). 자동 스크롤.
- 결과 영역: 완료 시 답변 카드(강조 배경) + "복사" 버튼.
- 상태: idle / running(스피너+단계수) / done / error / aborted — 칩으로 표시.

### Settings
- 제공자 선택: SegmentedButton [Anthropic][OpenAI호환][Gemini].
- API Base URL(OpenAI호환 시 노출, 기본값 채움).
- API Key: 비밀 입력(마스킹, 표시 토글). 저장 시 암호화 안내 캡션.
- 텍스트 모델명 입력(제공자별 기본 placeholder).
- 비전 모델명 입력(선택, 비우면 VLM 폴백 비활성).
- 최대 단계 수 슬라이더(5~40, 기본 20).
- "연결 테스트" 버튼 → 진행/성공/실패 인라인 표시.
- "저장" 버튼(스낵바 확인).

## 상태/빈/에러 표현
- 로딩: 진행 단계 스트림 자체가 로딩 표현. 초기엔 스피너+"준비 중".
- 빈: 입력 전 예시 칩과 안내 문구.
- 에러: 빨강 카드 + 사유 + 재시도/설정 이동 버튼.
- 오프라인/키오류: 명확한 사유 + 설정 유도.

## 카피(한국어 1인칭 보조자 톤)
- 실행 시작: "작업을 시작할게요."
- 폴백: "화면을 이미지로 살펴볼게요."(비전 사용 시)
- 완료: 요약 답변 그대로.
- 권한 필요: "화면을 읽고 조작하려면 접근성 권한이 필요해요."
