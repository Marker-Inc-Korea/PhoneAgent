# PhoneAgent — 기획서 (Concept)

## 한 줄 정의
안드로이드 접근성 서비스(AccessibilityService)로 **모든 앱의 화면을 텍스트로 읽고 조작**하는
온디바이스 LLM 에이전트. 텍스트 트리로 불충분하면 **스크린샷 + VLM**으로 폴백한다.

## 배경 / 문제
- 스마트폰 자동화는 앱별 API(Gmail API 등)에 의존하면 앱마다 연동을 새로 해야 한다.
- 사람이 쓰는 방식 그대로(화면을 보고, 누르고, 입력) 동작하면 **모든 앱**이 대상이 된다.
- OpenClaw(구 Clawdbot)류 데스크톱 에이전트의 접근성-우선 전략을 모바일로 가져온다:
  1. **1차: 접근성 트리** — 토큰 효율이 높고 빠르며 텍스트 LLM만으로 동작.
  2. **2차: 스크린샷 + VLM** — WebView/게임/캔버스 등 트리가 빈약한 화면 대응.

## 목표 (v1)
1. 사용자가 자연어로 작업을 지시하면 에이전트가 폰을 직접 조작해 수행한다.
2. **대표 시나리오**: "Gmail에서 새 이메일 확인하고 요약해줘" — Gmail 앱을 열고,
   받은편지함을 읽고, 새 메일 목록·내용을 한국어로 요약 보고한다.
3. LLM/VLM 제공자·모델·API 키를 앱 설정 화면에서 사용자가 직접 구성한다.
4. 사용자 폰에 바로 설치 가능한 APK로 제공한다.

## 비목표 (v1에서 제외)
- 온디바이스 모델 추론(클라우드 API만 사용)
- 멀티태스크 스케줄링, 백그라운드 정기 실행
- 결제·송금 등 고위험 작업의 자동 수행(에이전트는 가능하지만 안전 가이드로 억제)
- 루팅/ADB 권한 요구 — 일반 사용자 설치 경로(접근성 권한만)로 한정

## 핵심 전략
| 레이어 | 선택 | 이유 |
|---|---|---|
| 화면 읽기 | AccessibilityService 노드 트리 → 압축 텍스트 | 토큰 효율, 정확한 요소 식별(id 매핑) |
| 화면 조작 | performAction + dispatchGesture | 클릭 불가 노드도 좌표 제스처로 폴백 |
| 시각 폴백 | takeScreenshot(API 30+) + VLM | 트리 빈약 화면(노드<3) 자동 감지 + LLM이 스스로 요청 |
| 모델 연동 | Anthropic / OpenAI 호환 / Gemini, JSON 액션 프로토콜 | 제공자 중립, 툴콜 방언 회피 |
| 신뢰 경계 | API 키 Keystore 암호화, 키는 화면에 마스킹 | 자격증명 보호 |

## 성공 기준
- 시나리오 테스트(가짜 Gmail 화면 + 스크립트 LLM)로 에이전트 루프가 끝까지 완주한다.
- 단위 테스트 전체 통과, `assembleDebug`/`assembleRelease` 빌드 성공.
- 설치 후 사용자는: ① 접근성 권한 허용 ② API 키 입력 ③ 작업 지시 — 3단계로 사용 시작.
