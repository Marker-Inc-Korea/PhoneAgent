# PhoneAgent — 시나리오 정의

## S1. (대표) Gmail 새 이메일 요약
**사용자**: "Gmail에서 새로 온 이메일 확인하고 어떤 게 왔는지 설명해줘"

1. 에이전트가 `open_app("Gmail")` → Gmail 패키지(`com.google.android.gm`) 실행.
2. 받은편지함 스냅샷 수집 — 리스트의 보낸사람/제목/미리보기/읽음상태 텍스트 노드 확보.
3. 읽지 않은 메일 식별(굵은 표시 텍스트는 트리에서 동일 — 내용 기반 판단 + 필요 시 메일 개별 열람).
4. 필요하면 `tap(id)`으로 개별 메일 열어 본문 일부 확인 후 `global(back)`.
5. `scroll(down)`으로 추가 메일 확인(최대 N회).
6. `done(answer)` — 한국어 요약: 보낸사람·제목·핵심내용·중요도.

**완료 조건**: 사용자 화면(앱 + 알림)에 요약 텍스트가 표시된다.
**실패 처리**: Gmail 미설치 → `abort("Gmail 앱을 찾을 수 없음")`; 로그인 화면 감지 → 요약 대신 상태 보고.

## S2. VLM 폴백
1. 스냅샷의 의미 있는 노드가 3개 미만(예: 전체 WebView/캔버스) → 자동으로 스크린샷 촬영,
   다음 LLM 요청에 이미지 첨부(비전 모델 사용).
2. LLM이 텍스트 트리만으로 판단 불가하다고 보고 `screenshot` 액션을 반환 → 동일 폴백.
3. 비전 모델 미설정 시: 폴백 불가 사유를 step 로그에 남기고 텍스트로 계속 진행.

## S3. 설정
1. 설정 화면에서 제공자(Anthropic/OpenAI 호환/Gemini) 선택, API 키 입력, 텍스트 모델/비전 모델명 입력.
2. 저장 시 API 키는 Android Keystore로 암호화되어 DataStore에 저장.
3. "연결 테스트" 버튼 → 1-토큰 호출로 키/모델 유효성 확인 후 성공/실패 표시.

## S4. 온보딩/권한
1. 첫 실행: 접근성 권한 미허용 배너 → 버튼 탭 → 시스템 접근성 설정으로 이동.
2. 알림 권한(API 33+) 요청 — 실행 상태 알림용.
3. 권한 완료 후 홈 화면에서 작업 입력 가능 상태.

## S5. 실행 중 제어
1. 실행 중 화면 위 오버레이 상태 바(TYPE_ACCESSIBILITY_OVERLAY)에 현재 단계 표시 + 중지 버튼.
2. 중지 → 루프 취소, "사용자 중지" 결과 기록.
3. 최대 단계 수(기본 20) 초과 → 자동 중단 + 사유 보고.

## 엣지 케이스 (구현 전 열거 — 발생 가능성 상위 3개 ★)
- ★ LLM이 JSON 외 산문/마크다운 펜스 포함 응답 → 관용 파서(첫 균형 JSON 추출) + 재요청 1회.
- ★ 스냅샷-실행 사이 화면 변경(스테일 노드) → 최신 스냅샷의 id만 유효, 실패 시 좌표 제스처 폴백, 실행 후 안정화 대기.
- ★ Gmail 받은편지함 노드 수백 개 → 노드 상한(120) + 텍스트 절단(80자) + 비가시 노드 제외.
- API 429/5xx/타임아웃 → 지수 백오프 2회 재시도 후 단계 실패 보고.
- API 키 미설정 상태로 실행 → 실행 전 프리플라이트로 차단, 설정 화면 유도.
- 빈 작업 입력 / 초장문 입력 → 입력 검증(공백 차단, 2,000자 제한).
- 실행 중 화면 꺼짐/잠금 → 스냅샷의 keyguard 감지 시 보고 후 중단.
- 접근성 서비스가 도중에 꺼짐 → DeviceController 미연결 감지, 즉시 중단 + 안내.
- 같은 화면 반복(루프) → 동일 액션 3연속 감지 시 LLM에 경고 주입.
- 텍스트 입력 대상이 비편집 노드 → set_text 실패 보고를 LLM에 피드백.
- 이미지 크기 과대(고해상 스크린샷) → JPEG 품질/해상도 축소(긴 변 1280px).
- 한국어/이모지/RTL 텍스트 → 전 구간 UTF-8, 절단은 코드포인트 경계.
